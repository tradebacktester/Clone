## Cache Statistics

The engine queries `pg_statio_user_tables` to measure buffer cache efficiency:

| Metric | Formula | Target |
|--------|---------|--------|
| Heap Hit Ratio | heap_blks_hit / (heap_blks_hit + heap_blks_read) | ≥ 95% |
| Index Hit Ratio | idx_blks_hit / (idx_blks_hit + idx_blks_read) | ≥ 95% |

**Replit PostgreSQL** defaults work well for the current data volume. For production at scale (1M+ records), `shared_buffers = 256MB` or higher is recommended.

## Performance Score Calculation

```
performanceScore = benchmarkScore + cacheScore + latencyBonus

benchmarkScore   = (passing / total) × 60
cacheScore       = ((heapHitRatio + idxHitRatio) / 2) × 30
latencyBonus     = count(durationMs < target/2) × 2

Max score = 100
```

## Recommendations Generated by Engine

When benchmarks fail, the engine automatically generates specific SQL:

```sql
-- Slow compound filter (pair + outcome + session)
CREATE INDEX CONCURRENTLY idx_exp_compound
  ON memory_experiences (pair, outcome, session);

-- Slow timeline reconstruction
CREATE INDEX CONCURRENTLY idx_events_timeline
  ON trade_events (trade_id, event_type, occurred_at);
```

These are provided as recommendations, not auto-applied, to avoid blocking production.
